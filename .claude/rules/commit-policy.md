# Политика commit

## Commit допустим
Только если пользователь явно попросил выполнить commit.

## Перед commit
1. Убедись, что реализованы все шаги согласованного плана.
2. Выполни релевантные проверки.
3. Кратко зафиксируй, что именно войдёт в commit.
4. Получи явное подтверждение на commit, если оно ещё не было дано в текущем запросе.

## После commit
После успешного commit должен автоматически выполниться `graphify update .`.

## Запрещено
- делать push;
- делать release-команды;
- выполнять commit с несогласованным дополнительным объёмом изменений.
