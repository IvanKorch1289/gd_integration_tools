# Политика внешнего исследования

## Когда идти в интернет
Использовать web search / web fetch только если задача требует:
- актуальной документации библиотек;
- release notes / changelog;
- breaking changes;
- новых паттернов разработки;
- свежих примеров интеграции;
- сравнения современных инструментов.

## Доступные инструменты
- **DuckDuckGo MCP** (`duckduckgo`) — основной поиск, без API key, без third-party tracking.
- **Fetch MCP** (`fetch`) — официальный Anthropic-сервер для получения веб-страниц.
- **WebSearch / WebFetch** — встроенные инструменты Claude Code как fallback.

## Порядок
1. Сначала проверить внутренний контекст проекта через Graphify и docs.
2. Затем коротко сформулировать, чего именно не хватает.
3. Выполнить узкий поиск через DuckDuckGo MCP по конкретной библиотеке/паттерну.
4. При необходимости использовать Fetch MCP / WebFetch только для 1–2 наиболее релевантных источников.
5. В ответе явно отделять:
   - внутренний контекст проекта;
   - внешние актуальные данные (с источником и датой, если применимо).

## Источники по умолчанию
Предпочитать:
- официальную документацию;
- GitHub releases / changelog;
- блоги вендоров;
- авторитетные engineering-статьи.

Избегать:
- случайных SEO-статей;
- устаревших gist без контекста;
- анонимных непроверяемых источников.

## Если поиск недоступен
Явно сообщить пользователю: «ответ без актуальной внешней проверки» — не маскировать ограничение.
