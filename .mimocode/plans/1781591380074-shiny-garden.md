## АРХИТЕКТУРНЫЕ ПРАВИЛА (никогда не нарушать)

1. **Layered Architecture**: core → infrastructure → services → entrypoints
   - core НЕ импортирует services/infrastructure/entrypoints (только Protocol)
   - infrastructure НЕ импортирует services/entrypoints
   - services НЕ импортирует entrypoints
   - extensions импортируют ТОЛЬКО core/ + capability-checked фасады

2. **Facade-First**: каждый домен имеет facade в services/
   - Storage → StorageFacade (уже есть)
   - Cache → CacheFacade (создать)
   - EventBus → EventBusFacade (есть, расширить)
   - ExternalDB → ExternalDBFacade (создать)

3. **DSL-First**: весь функционал доступен через DSL builders
   - Если процессор есть, но builder-метода нет — это gap, исправить
   - Если builder-method есть, но processor — stub, это P0 bug

4. **Zero Tech Debt**: не переносить техдолг на следующий спринт
   - Каждый спринт закрывает ≥1 P0 + ≥2 P1 проблему
   - Sprint заканчивается review и changelog

1. `make lint` — должен быть clean
2. `make type-check` — 0 новых ошибок
3. `make test` — все тесты зелёные

5. Self-review checklist:
   - [ ] Protocol в core, impl в infrastructure/services
   - [ ] No direct imports across layer boundaries
   - [ ] DSL builder method added (if new processor)
   - [ ] Docstring updated
   - [ ] Test coverage for new code

### После каждого спринта:
1. CHANGELOG.md обновлён
2. ADR создан/обновлён (если архитектурное решение)

## ФАСАДЫ — ПРАВИЛА СОЗДАНИЯ

При добавлении нового facade:

1. Protocol определяется в `core/interfaces/<domain>.py`
2. Implementation生活在 `infrastructure/<domain>/`
3. Facade生活在 `services/<domain>/facade.py`
4. Facade имеет:
   - `__init__(self, backend: Protocol, *, capability_check, plugin)`
   - Capability-checked методы (publish/subscribe pattern)
   - Exception handling → ServiceError
5. Facade регистрируется в `plugins/composition/service_setup.py`
6. Tests: `tests/unit/services/<domain>/test_facade.py`

Шаблон facade:
```python
class DomainFacade:
    def __init__(self, backend: DomainProtocol, *, capability_check=None, plugin="extension"):
        self._backend = backend
        self._check = capability_check
        self._plugin = plugin

    def _assert(self, action: str, resource: str) -> None:
        if self._check:
            self._check(self._plugin, action, resource)

    async def do_something(self, resource: str, data: Any) -> Result:
        self._assert("domain.action", resource)
        try:
            return await self._backend.do_something(resource, data)
        except Exception as exc:
            raise ServiceError(f"domain action failed: {exc}") from exc
```

## АГЕНТСКИЙ ДОМЕН — ПРАВИЛА

1. **Tool permissions**: каждый агент имеет whitelist/blacklist tools в `AgentSpec.tools`
2. **Execution isolation**: агенты выполняются через `CodeSandbox` (E2B), не в main process
3. **Prompt caching**: использовать `AnthropicPromptCacheMiddleware` для эфемерного кэша
4. **Memory scoping**: `MemoryScope(mode="scoped")` — агент не видит чужую память
5. **Handoff limits**: `HandoffPolicy(max_handoffs=3)` — защита от infinite agent loops
6. **Policy enforcement**: `AIPolicyEnforcer` проверяет input/output guards
