## Lessons learned (для sprint-execution skill)

1. **Duplicate function definitions preserved in decomp** (S68 W3 NEW): when the original file has duplicate function definitions (e.g., `_serialize_request` defined twice with identical bodies), the decomp should preserve them as-is. Python's behavior (2nd shadows 1st) is preserved. If you want to flag this as a bug, do it in a separate fixup commit, not in the decomp.

2. **Per-provider file split pattern** (S68 W4 NEW): for pluggable provider classes (ClaudeProvider, GeminiProvider, etc.) with similar shape but different implementations, per-class file split is the right pattern. Each provider is self-contained, easy to find, easy to add new ones.
