# Политика решений по зависимостям

## Перед добавлением или обновлением зависимости
1. Проверить, есть ли уже аналог в stdlib или в текущих зависимостях проекта.
2. Проверить актуальную документацию и changelog.
   - Для API/синтаксиса/конфигурации — **Context7 MCP** (`mcp__context7__resolve-library-id` → `mcp__context7__query-docs`).
   - Для статуса проекта, сравнений, миграционного опыта — **DuckDuckGo MCP** (`mcp__duckduckgo__search`).
3. Оценить:
   - зрелость библиотеки;
   - частоту обновлений (last release date);
   - совместимость с Python 3.14;
   - сложность внедрения;
   - влияние на архитектуру.
4. Не предлагать обновление или новую зависимость без краткого сравнения альтернатив.
5. В отчёте Context7-цитаты помечать `[ctx7: <library>@<version>]`, веб-источники — с URL и датой.
