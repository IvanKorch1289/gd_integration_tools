## ОТЧЁТ ПО ДОМЕНУ: entrypoints/

**СТАТУС**: PARTIAL (3 критических дефекта)

### А. Что существует [ТОЧНО]
- **190+ Python файлов** в 20 поддиректориях
- **Unified dispatch**: `base.py::dispatch_action()` — единая точка входа для всех протоколов
- **Action bridge**: `_action_bridge.py` — мост между ActionDispatcher и DSL fallback
- **Middleware registry**: 25+ ASGI middleware с 4-уровневым ordering
- **Auth**: Defense-in-depth через `AuthRequiredMiddleware` (6 методов)

### Б. Что подключено и работает end-to-end [ТОЧНО]
- **REST API**: 40+ роутеров, OpenAPI auto-generated ✅
- **gRPC**: 22 файла + 10 .proto ✅
- **GraphQL**: Strawberry schema + auto-schema ✅
- **WebSocket**: Handler + manager + auth + broadcast ✅
- **SSE**: Stream + invoke ✅
- **MCP**: Gateway + server + namespaces ✅
- **Rate limiting**: 4-уровневая система ✅
- **Auth**: 6 методов + protocol-specific auth ✅

### В. Что является scaffold / мёртвым / сломанным [ТОЧНО]
1. **CRITICAL: Syntax error** — `websocket/ws_auth.py:91`: `except ImportError, AttributeError, ConnectionError:` — Python 2 syntax
2. **HIGH: Dead code** — `grpc/grpc_server.py` (510 LOC) — дубликат decompacted package
3. **MEDIUM: Deprecated shim** — `api/dependencies/auth_selector.py` — помечен на удаление S99+
4. **LOW: Duplicate APIVersion** — `middlewares/versioning.py` vs `api/versioning.py`

### Г. Что нарушает архитектурные правила [ТОЧНО]
- **Правило 2**: ~35+ imports entrypoints → infrastructure/services (lazy, но architectural)
- **Самые проблемные**: 8+ direct imports из infrastructure.scheduler в admin_cron.py

### Д. Реализован ли DSL [ТОЧНО]
- **Да**: Все entrypoints используют unified dispatch через `action_handler_registry`
- **DSL fallback**: Tier 1/2 (ActionDispatcher) → Tier 3 (DSL route) через `_action_bridge.py`
