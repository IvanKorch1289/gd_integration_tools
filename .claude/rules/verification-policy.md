# Политика самопроверки

## Принцип
После каждого изменения Claude обязан сам проверить результат минимально достаточными командами.

## Базовый порядок
1. Выбери минимальный релевантный набор проверок.
2. Запусти проверки.
3. Интерпретируй результат.
4. Если проблема вызвана твоим изменением — исправь её до следующего шага.
5. Не объявляй задачу завершённой без проверки.

## Примеры выбора проверок
- Изменения Python-кода: `make lint` или `make lint-strict`, затем `make type-check`
- Изменения public contracts / DI / DSL: дополнительно `make routes`, `make actions`
- Изменения документации/portal: `make docs`
- Изменения зависимостей/секретов/релизных вещей: `make deps-check`, `make secrets-check`, `make readiness-check`

## Запрещено
- запускать всё подряд без необходимости;
- пропускать проверки “потому что изменение маленькое”;
- игнорировать ошибки, связанные с внесённым изменением.
