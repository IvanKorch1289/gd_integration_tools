## A. EXECUTIVE SUMMARY — 20 ключевых выводов

### CRITICAL (P0) — Немедленно

1. **`except A, B:` — Python 3.14 semantic bug в 77 файлах (~95 мест)**. Конструкция `except ValueError, KeyError:` ловит только `ValueError`, а `KeyError` становится именем переменной. КРИТИЧНО для банковской системы. **Файл-пример**: `core/auth/jwt_blacklist.py:144`, `infrastructure/database/initializer.py:211`, `dsl/engine/execution_engine.py:208`. **Инструмент исправления уже есть**: `tools/fix_except_bug.py`.

2. **SSL verification отключена глобально** — `config_profiles/base.yml:131` — `ssl_verify: false`. Ни один overlay (prod/staging) НЕ переопределяет в `true`. ВСЕ HTTP outbound вызовы пропускают проверку TLS-сертификатов. Для банка — критический риск MITM.

3. **InProcessAgentSandbox остаётся default** — `services/ai/agent_sandbox.py:85` + `dsl/engine/processors/agent_dsl/agent_graph.py` — агентный код выполняется в том же процессе без изоляции. Компрометация агента = полный доступ к процессу.

4. **3 файла с pickle.loads** — `infrastructure/database/query_result_cache.py:74`, `dsl/engine/processors/format_convert/data_formats.py:247`, `dsl/engine/processors/eip/marshal/formats.py:269`. Если Redis/файл скомпрометирован — RCE.

5. **GraphQL endpoint без auth** — `entrypoints/graphql/schema.py` — нет `require_auth`, нет `Depends`. Полный доступ к данным без аутентификации.

6. **WebSocket без auth** — `entrypoints/websocket/ws_handler.py` — `ws_auth.py` написан (128 строк) но НЕ используется ни одним handler'ом.

7. **DSL Console — arbitrary code execution** — `entrypoints/api/v1/endpoints/dsl_console.py:22-36` — принимает произвольный YAML pipeline и выполняет без auth. Через DSL processors (script_runner, ssh_exec, sql_exec) — полноценный RCE.

8. **Admin endpoints без auth** — `entrypoints/api/v1/endpoints/admin_plugins.py` — только feature-flag защита, нет require_auth.

### HIGH (P1) — 1-2 недели

9. **33 frontend → backend прямых импорта** — frontend не является thin client. Страницы (54_DLQ_Replay, 32_DSL_Builder, 34_DSL_Debugger, 62_Schema_Admin и др.) напрямую импортируют `src.backend.*`.

10. **~50 файлов DSL → infrastructure прямых импортов** — нарушение слоистости. DSL engine_processors тянут redis, S3, http-клиенты напрямую вместо DI.

11. **~18 файлов core → infrastructure прямых импортов** — `core/dsl/variables.py`, `core/cdc/registry.py`, `core/resilience/cache_decorators.py` и др.

12. **Vault dev mode в production compose** — `ops/compose/docker-compose.prod.yml:165` — `-dev` flag отключает TLS и использует in-memory storage.

13. **Admin login — silent mock token fallback** — `entrypoints/api/v1/endpoints/auth_login.py:183-186` — при ошибке JWT генерирует `mock-jwt-{username}` без проверки environment.

14. **Composition route ссылается на несуществующие файлы** — `routes/composition_demo/main.dsl.yaml` — `./base_transforms.yaml` и `./shared_steps.yaml` отсутствуют.

15. **Test route ссылается на несуществующий модуль** — `routes/test_route_w1/main.dsl.yaml` — `extensions.test_route_w1.normalizer:apply_rules` не существует.

16. **Placeholder paths в production routes** — `routes/hello_route/` и `routes/test_route_w1/` — `/api/v1/CHANGEME`.

### MEDIUM (P2) — 2-4 недели

17. **SOAP/GraphQL exception details leak** — `entrypoints/soap/soap_handler.py:207` — `str(exc)` в SOAP fault раскрывает stack traces и connection strings клиентам.

18. **WSAuthenticator написан но не используется** — `entrypoints/websocket/ws_auth.py` — 128 строк мёртвого кода.

19. **SAML cookie — plain text principal** — `entrypoints/api/v1/endpoints/auth_saml.py:173` — cookie содержит `principal|session_index` в открытом виде вместо opaque session ID.

## C. DOMAIN SUMMARIES

### 1. CORE (`src/backend/core/`)

**Назначение**: Domain-agnostic kernel — protocols, interfaces, DI, resilience, security, tenancy.
**Основные сущности**: AIGateway, AgentRegistry, CapabilityGate, CircuitBreaker, TenantContext, PluginLoader.
**Зрелость**: HIGH.
**Что хорошо**: Protocol-based interchangeability (PEP 544), capability-gated features, resilience patterns.
**Проблемы**: 18 файлов с прямым импортом infrastructure (core→infra violation), facade re-export shims (14 файлов), deprecated WorkflowBuilder всё ещё используется.
**Оценка**: 7/10.

### 2. INFRASTRUCTURE (`src/backend/infrastructure/`)

**Назначение**: External integrations — databases, caches, messaging, storage, workflows.
**Основные сущности**: DatabaseManager, RedisClient, S3ObjectStorage, TemporalWorkflowBackend, VaultClient.
**Зрелость**: HIGH.
**Что хорошо**: Connection pooling везде, path traversal protection, SQL injection protection.
**Проблемы**: pickle.loads (3 файла), presidio_sanitizer.py legacy shim, bulk operations без batch limits, `except A, B:` в 20+ файлах.
**Оценка**: 7/10.

### 3. DSL (`src/backend/dsl/`)

**Назначение**: Declarative route/workflow definition — YAML + Python builder.
**Основные сущности**: ExecutionEngine, RouteBuilder (god-class, 150+ методов), Pipeline, BaseProcessor (~50+ процессоров).
**Зрелость**: MEDIUM-HIGH.
**Что хорошо**: 80/20 YAML/Python split, EIP patterns, hot-reload с SHA-256 caching, middleware chain.
**Проблемы**: 50+ файлов с infrastructure imports, RouteBuilder god-class (10-mixin MRO), `except A, B:` в 18 файлах, duplicate `_DEFAULT_TEMPERATURE` в 5 файлах.
**Оценка**: 7/10.

### 4. SERVICES (`src/backend/services/`)

**Назначение**: Business services — AI, health, integrations, ops, plugins, routes.
**Основные сущности**: AIAgentService, RAGService, OrderService, NotebookService, AdminService.
**Зрелость**: MEDIUM-HIGH.
**Проблемы**: InProcessAgentSandbox default, tool whitelist bypass, guard failures return "passed".
**Оценка**: 6.5/10.

### 5. ENTRYPOINTS (`src/backend/entrypoints/`)

**Назначение**: Protocol adapters — REST, GraphQL, gRPC, SOAP, WebSocket, SSE, MQTT, MCP, CDC.
**Основные сущности**: FastAPI app, GraphQL schema, gRPC server, SOAP handler, WS handler.
**Зрелость**: MEDIUM.
**Что хорошо**: SSE имеет auth, MCP auto-export, middleware pipeline (26 ASGI MW).
**Проблемы**: GraphQL без auth, WebSocket без auth (ws_auth не подключён), DSL Console arbitrary execution, admin endpoints без auth, mock token fallback, exception leaks в SOAP/SSE.
**Оценка**: 5.5/10.

### 6. FRONTEND (`src/frontend/streamlit_app/`)

**Назначение**: Operator dashboard + documentation (36+ pages).
**Зрелость**: MEDIUM.
**Что хорошо**: Good API client abstraction (12 domain clients), Streamlit session state, secret masking.
**Проблемы**: 33 прямых `src.backend` импорта, BaseAPIClient создаёт httpx.Client на каждый запрос (no connection reuse), admin-react node_modules в репозитории.
**Оценка**: 5/10.

### 7. EXTENSIONS (`extensions/`)

**Назначение**: Business logic plugins.
**Зрелость**: MEDIUM.
**Что хорошо**: Well-structured plugin manifest, lifecycle hooks, topological sort.
**Проблемы**: 3 разных import path для BasePlugin, inconsistent plugin.toml format (`[plugin]` header vs bare keys), version constraint drift (`>=0.2,<0.3` vs `>=22.0,<23`), example_plugin использует legacy paths.
**Оценка**: 6/10.

### 8. AI / AGENTS

**Назначение**: LLM gateway, multi-agent orchestration, RAG, guardrails.
**Зрелость**: MEDIUM.
**Что хорошо**: 9-step pipeline, AI workspace isolation, LangFuse observability, capability-gated tools.
**Проблемы**: InProcessAgentSandbox default, tool whitelist на workflow_id вместо actual tool, module whitelist skipped, guard failures return "passed", policy references removed mem0ai.
**Оценка**: 5/10.

### 9. WORKFLOW / ORCHESTRATION

**Назначение**: Durable workflow execution — Temporal/Lite/pg-runner.
**Зрелость**: MEDIUM-HIGH.
**Что хорошо**: 3 backend'а (Temporal/Lite/pg-runner), advisory lock, event replay, LISTEN/NOTIFY.
**Проблемы**: pg_runner busy-wait polling, replay() no-op, WorkflowBuilder deprecated но используется.
**Оценка**: 7/10.

### 10. CONFIGURATION / DEPLOYMENT

**Зрелость**: MEDIUM.
**Что хорошо**: YAML profiles, Pydantic settings, Vault integration, Helm chart с HPA/PDB.
**Проблемы**: SSL disabled globally, Vault dev mode в prod compose, hardcoded IPs в dev config, JWT HS256, dual doc generators (mkdocs + Sphinx), admin routes без API key по умолчанию.
**Оценка**: 5.5/10.

### 11. DOCUMENTATION

**Зрелость**: MEDIUM-HIGH.
**Что хорошо**: Diataxis structure, 190+ ADRs, audit reports, runbooks, tutorials.
**Проблемы**: dual generators (mkdocs + Sphinx), some tutorial path mismatches, no docstring linter в CI.
**Оценка**: 7/10.

### 12. TESTS

**Зрелость**: MEDIUM.
**Что хорошо**: 1,450 test files, 25 chaos tests, testkit with fixtures, VCR-style cassettes.
**Проблемы**: E2E coverage minimal (1 файл), no integration tests for extensions, no ai_policies validation tests.
**Оценка**: 6/10.

### What is Already Good (Do Not Break)

1. Protocol-based interchangeability (PEP 544)
2. Capability-gated features throughout
3. Resilience patterns (CB, retry, bulkhead, rate-limit)
4. Multi-backend support (Temporal, Lite, pg-runner)
5. DSL/YAML declarative approach с hot-reload
6. Connection pooling для всех external clients
7. Path traversal protection в file operations
8. SQL injection protection via parameterized queries
9. SSE auth coverage
10. Helm chart с HPA/PDB/network policy

### What Must Be Isolated Before Scaling

1. **Agent sandbox** — InProcessAgentSandbox must be deprecated
2. **Tool policy enforcement** — must be wired to actual tool invocation
3. **Auth coverage** — GraphQL, WS, DSL Console, Admin
4. **SSL verification** — must be enabled in production

### What is Dangerous to Ship to Prod Now

1. **SSL verification disabled** — MITM risk for all HTTP calls
2. **Zero-isolation agent sandbox** — agent compromise = full process access
3. **GraphQL/WS without auth** — unauthorized data access
4. **DSL Console without auth** — arbitrary code execution
5. **pickle.loads** — RCE if Redis/files compromised
6. **Mock token fallback** — could leak to production
7. **77 файлов с `except A, B:`** — silently swallowing exceptions

### What Can Become Stable Public API for Extensions

1. `core.api` — Facade для AI, Auth, Workflow, Config
2. `BasePlugin` lifecycle hooks (единый import path)
3. `ProcessorRegistryProtocol` для DSL processors
4. `ActionRegistryProtocol` для actions
5. `CapabilityGate` для capability declaration
