```python
# src/backend/entrypoints/lifespan.py (предложить PR)
from src.backend.infrastructure.security.cert_store import (
    CertStore, CertFileWatcher,
)

@asynccontextmanager
async def lifespan(app):
    store = CertStore.from_settings(cert_store_settings)
    app.state.cert_store = store

    if cert_store_settings.watch_enabled:
        watcher = CertFileWatcher(
            path=Path(cert_store_settings.watch_path),
            store=store,
        )
        await watcher.start()
        app.state.cert_watcher = watcher

    yield

    if hasattr(app.state, "cert_watcher"):
        await app.state.cert_watcher.stop()
```

### TASK-2: Удалить mock-JWT fallback в `auth_login.py`

**Файл:** `src/backend/entrypoints/api/v1/endpoints/auth_login.py:182-194`

**Проблема:** При ошибке `jwt_encode` и `env == "development"` код возвращает `f"mock-jwt-{user}-{ts}"`. Mock-токен может быть принят middleware в misconfigured prod (где `environment` ошибочно выставлен в development). Это безопасный dev-инструмент, который становится бэкдором.

**Fix:**
```python
# Before (lines 182-194):
except (TypeError, ValueError) as exc:
    ...
    if _env != "development":
        raise HTTPException(500, "Token generation failed") from exc
    _logger.warning(...)
    token = f"mock-jwt-{user.username}-{int(time.time())}"

# After:
except (TypeError, ValueError) as exc:
    _logger.error("auth.login.jwt_encode_failed err=%s env=%s", exc, _env)
    raise HTTPException(500, "Token generation failed") from exc
```

JWT encode failure = всегда 500. Dev может сбросить secret через Vault. Никаких mock-fallback'ов.

**Effort:** XS (5 минут). **Risk:** LOW — dev workflow меняется (нужно валидное JWT secret в dev).

### TASK-3: Заменить `pickle.loads` в query_result_cache.py

**Файл:** `src/backend/infrastructure/database/query_result_cache.py:74`

**Проблема:** Кеш читает с диска/Redis через `pickle.loads(data)`. Если кеш-стор скомпрометирован (Redis injection / disk write) — RCE в backend-процессе.

**Fix:** Заменить на `json.loads` + `msgpack.loads` (msgpack уже в deps). Структура: msgpack для binary-эффективности, fallback на json для dict/list.

Ponytail: добавить helper `_deserialize(data: bytes) -> Any`:
```python
def _deserialize(data: bytes) -> Any:
    try:
        return msgpack.unpackb(data, raw=False, strict_map_key=False)
    except (msgpack.exceptions.UnpackException, ValueError):
        return json.loads(data.decode("utf-8"))
```

Использовать в `query_result_cache.py` и убедиться, что в write-path используется соответствующий `_serialize`.

**Effort:** S (30 минут). **Risk:** LOW — только изменение сериализации, контракт тот же.

### TASK-4: Audit/remove `pickle.loads` в EIP marshal `formats.py`

**Файл:** `src/backend/dsl/engine/processors/eip/marshal/formats.py:269`

**Проблема:** EIP marshal step десериализует caller-provided bytes через pickle. Если кто-то передаёт untrusted bytes (e.g. через HTTP route) — RCE. Comment на строке 264 уже говорит об этом.

**Fix:** Заменить на `msgpack.unpackb` (symmetric к serialize). Удалить `pickle` импорт, удалить `noqa: S301`. Обновить соответствующий `serialize` путь. Добавить capability gate: marshal step требует `eip.marshal.privileged` capability (по умолчанию deny в проде).

Ponytail-lite: если EIP marshal — редко используемая фича, можно оставить pickle, но **добавить feature flag** `eip.marshal.pickle_enabled: bool = False` (default OFF). Это инвертирует риск: не включать pickle, пока явно не запросят.

**Effort:** S (20 минут). **Risk:** LOW — feature flag default OFF.

**Решение:** Default-OFF feature flag + capability gate — наименее invasive.

```python
# Before:
```
