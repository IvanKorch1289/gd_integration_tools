# Политика subagents

## Когда использовать
Subagents использовать для:
- исследований по документации;
- DSL-анализа;
- runtime-диагностики;
- contract review;
- dead code review;
- verification.

## Когда не использовать
- Для финального согласования с пользователем
- Для действий, требующих AskUserQuestion
- Для задач, где нужен общий пошаговый контроль над исполнением

## Правила
- Главный агент планирует и координирует
- Subagent получает одну узкую задачу
- Subagent возвращает короткий структурированный результат
- Не делегировать всё подряд, если проще сделать напрямую

## Multi-agent
- Для сложных независимых исследований можно запускать несколько subagents параллельно
- Для долгих координируемых задач можно использовать agent teams, но помнить, что это дороже по токенам
