# Sprint 20 — RPA Domain Critical Fixes

## Результаты глубокого аудита RPA домена

### Статус: CRITICAL ISSUES FOUND

### Ключевые находки

#### КРИТИЧЕСКИЕ ПРОБЛЕМЫ (3)

1. **Syntax error в browser_cookies_store.py:117** [ТОЧНО]
   - `except TypeError, json.JSONDecodeError:` — Python 2 синтаксис
   - В Python 3 это SyntaxError — модуль не может быть импортирован
   - **Исправление:** Заменить на `except (TypeError, json.JSONDecodeError):`

2. **Blocking I/O в filemoveprocessor.py:49-56** [ТОЧНО]
   - `shutil.move()`, `os.rename()`, `shutil.copy2()` вызываются напрямую в `async def process()`
   - Блокируют event loop на время I/O операции
   - **Исправление:** Обернуть в `asyncio.to_thread()`

3. **Encryption keys leaked в to_spec()** [ТОЧНО]
   - `encryptprocessor.py:54` и `decryptprocessor.py:49` включают Fernet ключ в сериализацию
   - Ключи видны в логах, admin UI, DSL каталоге
   - **Исправление:** Не включать ключ в to_spec(), использовать placeholder

#### ВЫСОКИЕ ПРОБЛЕМЫ (3)

4. **Dead code в ocr_processor.py** [ТОЧНО]
   - Весь модуль (155 строк) не используется — 0 импортов
   - **Исправление:** Удалить или подключить к DSL

5. **Dead code в desktop_session_pool.py** [ТОЧНО]
   - Pool class (256 строк) никогда не подключается
   - **Исправление:** Удалить или подключить к startup

6. **Jinja2 SSTI vulnerability в templaterenderprocessor.py:39** [ТОЧНО]
   - Используется `Template()` без sandbox
   - **Исправление:** Использовать `jinja2.sandbox.SandboxedEnvironment`

#### СРЕДНИЕ ПРОБЛЕМЫ (4)

7. **DSL coverage gap: RegexProcessor split/findall** [ТОЧНО]
   - Builder документирует `action="split"`, но processor не реализует
   - **Исправление:** Реализовать split/findall в RegexProcessor

8. **Zip Slip vulnerability в archiveprocessor.py:62-85** [ТОЧНО]
   - Извлечённые имена файлов не санитизируются
   - **Исправление:** Добавить проверку path traversal

9. **Layer violation в system.py:119** [ТОЧНО]
   - DSL processor импортирует из infrastructure
   - **Исправление:** Использовать Protocol из core/interfaces

10. **5 banking processors — property-only stubs** [ТОЧНО]
    - Citrix, Terminal3270, Appium, EmailDriven, KeystrokeReplay
    - Только устанавливают properties, не выполняют работу
    - **Исправление:** Документировать как intentional delegation stubs
