## ЧАСТЬ 1: ОТВЕТЫ НА ВОПРОСЫ

### ai_2026.py — зачем нужен, можно ли объединить

**Содержимое `ai_2026.py`** (383 LOC): LiteLLMGatewaySettings, RagCacheSettings, BGESettings, LangMemSettings, StreamingLLMSettings, AIAgentSettings, LangFuseSettings, McpSettings, RagIngestSettings.

**Содержимое `ai.py`** (343 LOC): PerplexitySettings, HuggingFaceSettings, OpenWebUISettings, OpenRouterSettings, NimSettings, OpenAISettings, MarkitdownSettings.

**Статус использования `ai_2026.py`**: 30+ импортов по всему проекту (services/ai/gateway/client.py, entrypoints/mcp/, services/ai/rag_ingest_service.py, main.py и др.).

**Вывод**: Файлы **комплементарны**, не дублируют друг друга:
- `ai.py` = провайдеры (Perplexity, HuggingFace, OpenWebUI, OpenRouter, NIM, OpenAI)
- `ai_2026.py` = AI stack (LiteLLM gateway, RAG cache, BGE embeddings, LangMem memory, streaming, LangFuse observability, MCP)

**Рекомендация**: Переименовать `ai_2026.py` → `ai_stack.py` (убрать год из названия). Объединять с `ai.py` **НЕЛЬЗЯ** — это нарушит единственность ответственности (провайдеры vs stack). Оба файла слишком большие (343+383=726 LOC), объединение создаст God File.

**Действие**: `mv ai_2026.py ai_stack.py` + mass rename импортов (30+ locations).

### v11.py — что это, полный rename V11 ecosystem

**Содержимое `v11.py`** (89 LOC): `V11Settings` — feature flags для PluginLoader и RouteLoader.

**Экосистема V11** (8 файлов с v11 в имени):

## ЧАСТЬ 2: OSINT AGENT EXTENSION

### Концепция

Расширение `extensions/osint_agent/` — DSL route, принимающий ИНН клиента, выполняющий web-search через Perplexity, и генерирующий строгий OSINT-отчёт (макс. 3000 символов) с:
- Общая информация о компании
- Позитивные упоминания
- Негативные упоминания / жалобы
- Участие в судебных делах
- Финансовые маркеры (если доступны)
- Источники (citations)

### Архитектура

```
extensions/osint_agent/
├── plugin.toml              # V11 manifest
├── plugin.py                # BasePlugin entry
├── domain/
│   ├── __init__.py
│   └── models.py            # OsintReport, CompanyInfo Pydantic-модели
├── functions/
│   ├── __init__.py
│   └── osint_workflow.py    # Ядро: search → parse → compose report
└── tests/
    ├── __init__.py
    └── test_osint_workflow.py

routes/osint_agent/
├── route.toml               # V11 route manifest
└── osint.dsl.yaml           # DSL pipeline: web_search → llm_call → response
```

### DSL pipeline (osint.dsl.yaml)

```yaml
route_id: osint_agent
source:
  http:
    method: POST
    path: /api/v1/osint/report
steps:
  # 1. Валидация ИНН
  - call_function:
      ref: extensions.osint_agent.functions.osint_workflow:validate_inn
      result_property: company_info

  # 2. Web-search через Perplexity (3 параллельных запроса)
  - web_search:
      engine: perplexity
      query_source: body.search_query_general
      max_results: 10
      to: body.results_general

  - web_search:
      engine: perplexity
      query_source: body.search_query_courts
      max_results: 10
      to: body.results_courts

  - web_search:
      engine: perplexity
      query_source: body.search_query_negative
      max_results: 10
      to: body.results_negative

  # 3. Compose prompt из результатов
  - call_function:
      ref: extensions.osint_agent.functions.osint_workflow:compose_prompt
      result_property: osint_prompt

  # 4. LLM call для генерации отчёта
  - llm_call:
      provider: perplexity
      model: sonar
      prompt_property: osint_prompt
      result_property: osint_report_raw

  # 5. Валидация формата
  - call_function:
      ref: extensions.osint_agent.functions.osint_workflow:validate_report
      result_property: osint_report

to:
  response:
    code: 200
    body: ${osint_report}
```

### Валидация ИНН

```python
def validate_inn(inn: str) -> bool:
    """Проверка ИНН (10 или 12 цифр, контрольные суммы)."""
    if not inn.isdigit() or len(inn) not in (10, 12):
        return False
    # Контрольная сумма для 10-значного ИНН
    weights_10 = [2, 4, 10, 3, 5, 9, 4, 6, 8]
    check_10 = sum(int(inn[i]) * weights_10[i] for i in range(9)) % 11 % 10
    if len(inn) == 10:
        return check_10 == int(inn[9])
    # Контрольная сумма для 12-значного ИНН
    weights_12 = [7, 2, 4, 10, 3, 5, 9, 4, 6, 8]
    check_11 = sum(int(inn[i]) * weights_12[i] for i in range(10)) % 11 % 10
    check_12 = sum(int(inn[i]) * weights_12[i-1] for i in range(1, 11)) % 11 % 10
    return check_11 == int(inn[10]) and check_12 == int(inn[11])
```

### Поисковые запросы

Для каждого ИНН генерируются 3 запроса:
1. **Общий**: `"ООО Название ИНН {inn} site:rusprofile.ru OR site:list-org.com"`
2. **Суды**: `"{ИНН} судебные дела Арбитражный court"`
3. **Негатив**: `"{НАИМЕНОВАНИЕ} отзывы жалобы проблемы"` или `"{НАИМЕНОВАНИЕ} мошенничество"` (осторожно)
