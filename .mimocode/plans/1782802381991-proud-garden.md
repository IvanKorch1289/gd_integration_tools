### M1.1 — WebSocket Auth facade (HIGH)

**Файл**: `src/backend/entrypoints/websocket/ws_auth.py` (УЖЕ существует, 122 LOC). Проверить его содержимое.

**Изменения**:
1. **Дополнить `ws_handler.py`**: перед `await ws_manager.connect(...)` вызвать auth через `_authenticate_ws(websocket, settings)`.
2. Auth strategy: **Sec-WebSocket-Protocol subprotocol**: client передаёт `Sec-WebSocket-Protocol: jwt.<token>`, server парсит, верифицирует через `auth_selector.jwt_verify()` / `auth_selector.api_key_verify()` / cookie-based session. **FALLBACK**: если subprotocol содержит `apikey.<key>` — API key. **FALLBACK**: cookie `auth_session=<jwt_token>`.
3. **По user answer**: facade со всеми тремя методами (JWT/API-key/cookie) с явным приоритетом: subprotocol → cookie (только если scope содержит session-aware routes).
4. Если ничего не передано — `1008 close code: missing_credentials` + audit event `ws.auth.rejected`.
5. Документировать в docstring + добавить knob `ws_settings.require_auth: bool = True` (default ON).

**Изменяемые файлы**:

### M1.3 — Tool-policy dedup (LOW)

**Файл**: `src/backend/core/ai/gateway_orchestrator_mixin.py` (161 LOC).

1. Extract helper `_enforce_tool_policy_once(self, request, policy)`:
   ```python
   def _enforce_tool_policy_once(self, request: AIRequest, policy: AIPolicySpec | None) -> None:
       """Single tool-policy enforcement point (S1 fix).
       
       Used once per invoke (replaces duplicate checks at step 1.5 and 5).
       """
       if policy is None:
           return
       tools = getattr(policy, "tools", None)
       if tools is None:
           return
       if not (tools.whitelist or tools.blacklist):
           return
       from src.backend.core.ai.policy.enforcer.tools_policy import enforce_tool_policy
       enforced_name = request.tool_name or request.workflow_id
       enforce_tool_policy(enforced_name, tools)
   ```
